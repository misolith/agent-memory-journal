# v0.2.1 Issue List (Post-v0.2.0 Refinement)

The following items were identified after the v0.2.0 3-tier architecture landed. Most of them shipped in v0.2.1 / v0.2.2 / v0.2.3 — see `CHANGELOG.md` for the per-release breakdown.

## Documentation
- [x] **docs/glossary.md**: Define terms like `pinned`, `supersedes`, `decay`, `ingest`, and `tier`.
- [x] **docs/open_questions.md**: Document architectural trade-offs (e.g., BM25 vs Vector) and unresolved design decisions.
- [ ] **Session Lifecycle**: Formally define when a session is considered "closed" and when `.memory/sessions/` files should be archived or deleted. (Still open — tracked in `docs/open_questions.md` §2.)

## Core Logic & Refinement
- [x] **Semantic Deduplication**: Promotion uses Jaccard + token-overlap matching (`agent_memory/promote.py`).
- [x] **Last Seen Metadata**: Core entries carry `last_seen` and `recall_core` updates it on every successful hit.
- [x] **Supersedes Validation**: `append_core_memory` now fails fast when `supersedes` points to a missing id.
- [x] **Recall Index Caching**: `BM25Index.from_cache` persists the index under `.memory/index/` and invalidates on `NORMALIZER_VERSION` bumps.

## CLI / UX
- [x] **Explicit Legacy Export**: `LegacyJournal` lives in `agent_memory.legacy` and is re-exported from `agent_memory`.
- [x] **Doctor Fixes**: `agent-memory-journal doctor --fix` regenerates the manifest. As of v0.2.3, the manifest is also refreshed on every legitimate `remember`/`forget`/recall, so `doctor` only flags real out-of-band edits.

## Maintenance
- [ ] **Outdated Content Audit**: Review `core/` files for remaining test data or placeholder notes that should be superseded or archived.
